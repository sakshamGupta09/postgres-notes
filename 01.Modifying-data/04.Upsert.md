# Upsert

- On inserting a new row into a table, sometimes we get an error like the row already exists because of some constraints.
- In that case, it will simply update the existing row instead of throwing an error.


```sql
INSERT INTO table (C1, C2, C3)
VALUES (v1, v2, v3)
ON CONFLICT target action
```

## Action

## Do nothing

- Do not throw an error if the column value already exists but don't insert anything as well.

``` sql
INSERT INTO customers (C1, C2, C3)
values (V1, V2, V3)
on conflict (C1)
do
	nothing;
```

## Update

- I want to take values passed in insert statement and update my existing columns.
- To read teh values from INSERT statement we use `excluded.column_name`.

```sql
INSERT INTO customers (C1, C2, C3)
values (V1, V2, V3)
on conflict (C1)
DO UPDATE
SET C1=EXCLUDED.C1, C2=EXCLUDED.C2;
```
