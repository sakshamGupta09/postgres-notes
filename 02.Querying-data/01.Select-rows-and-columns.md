# Read data from table

- A table has rows and each rows has columns.
- When we use `SELECT` without `where` clause it gives us all the rows.
- Now, for all those rows, we can control which columns do we want.



## Selecting all the columns

```sql
SELECT
  *
FROM table
```
- Go to this table. `From` caluse is evaluated first.
- Get me all the rows.
- For every row I need all the available columns.
- Not a good practice because mostly the consumer will be needing a subset of the columns, So, we shouln't be sending extra data in the network.
