# Read data from table

- A table has rows and each rows has columns.
- When we use `SELECT` without `where` clause it gives us all the rows.
- Now, for all those rows, we can control which columns do we want.



## Selecting all the columns

```sql
SELECT
  *
FROM table
```
- Go to this table. `FROM` caluse is evaluated first.
- Get me all the rows as there is no `WHERE` clause.
- For every row I need all the available columns. `SELECT` clause.
- Not a good practice because mostly the consumer will be needing a subset of the columns. If we use `*`, first DBMS will send us more data which means more network bandwidth and then we will send this useless data to client again wasting memory and bandwidth.


## Selecting specific columns

```sql
SELECT
 C1,
 C2,
 C3
FROM Table
```
