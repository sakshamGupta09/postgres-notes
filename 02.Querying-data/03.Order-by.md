# Order By

- To sort the data in our table by one or more columns.
- Default order is `ASC`

```sql
SELECT
  *
FROM Table
ORDER BY
  C1 [ASC | DESC],
  C2 [ASC | DESC]
```

## Order by Aliases

- First `From` runs, then `WHERE`, then `SELECT` and then `ORDER BY`. Hence, we can use aliases also for sorting.

```sql
SELECT
	 first_name,
 	last_name,
	 gender,
	 date_of_birth AS "dob"
FROM actors
ORDER BY
	 gender ASC,
	 dob DESC;
```

```sql
SELECT
	first_name,
	last_name,
	LENGTH(first_name) AS "length"
FROM actors
ORDER BY
	length DESC;
```

## Order by column number

- Order by runs after `select` and `where`. So, It always runs on a result set.
- Now instead of sorting by column names, we can instead sort by column order in the resulting set.
- If we change the order of columns, this flow will break.

```sql
SELECT
	first_name,
	last_name,
	LENGTH(first_name) AS "length"
FROM actors
ORDER BY
	1 ASC,
	3 DESC;
```

## How nulls are sorted

- By default, If we sort by a column that has `null` values, `null` values comes at the last. On sorting in DESC, `nulls` appear at last.
