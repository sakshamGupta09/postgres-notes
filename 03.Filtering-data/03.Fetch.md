# Fetch

- When a query returns very large results (thousands or millions of rows), it can lead to High memory usage, Slow network transfer and Application may only need a few rows at a time.
- Generally, we query a DBMS and it returns us the result set even if the result set is massive.


## Cursor

- A cursor is a pointer to the result set of a query that lets you read rows one at a time (or in small batches) instead of retrieving all rows at once.
- Without cursor, we would get all 1M rows in response to a query.
- With cursor, we get a pointer and can read rows incrementally.



## Steps

### 1.Start a transaction

```sql
BEGIN
```

### 2.Declare cursor

```sql
DECLARE movie_cursor CURSOR FOR
SELECT * FROM movies;
```

## Step 3 — Fetch rows

- After this the cursor moves and if we run this query again, it returns the next 5 rows.

```sql
FETCH 5 FROM movie_cursor;
```

## Step 4 — Close cursor

```sql
CLOSE movie_cursor;
COMMIT;
```
